## Concept alignment
- **Concept doc(s)**: [`concepts/phi-core-mapping.md`](../../../concepts/phi-core-mapping.md) §"agent_loop free function" (phi-core is the execution engine) + [`concepts/permissions/05-memory-sessions.md`](../../../concepts/permissions/05-memory-sessions.md) §"session lifecycle" (sessions run agents; transcripts are real execution output) + [`concepts/agent.md`](../../../concepts/agent.md) §"agent executes tasks" + [`concepts/system-agents.md`](../../../concepts/system-agents.md) §"memory extraction reads session transcripts" (s02 depends on real transcripts)
- **Concept claim (verbatim or close paraphrase)**: Sessions run agents via phi-core's execution primitives. The transcript that results is the record of real agent execution. Memory-extraction + agent-catalog listeners consume real transcript data.
- **Contradiction**: **Contradicts concept directly.** Shipped `spawn_replay_task` ([`launch.rs:342+`](../../../../../../modules/crates/server/src/platform/sessions/launch.rs#L342)) feeds a canned 4-event synthetic sequence (`AgentStart` → `TurnStart` → `TurnEnd` → `AgentEnd`) through `BabyPhiSessionRecorder` instead of calling `phi_core::agent_loop()`. The `use phi_core::{agent_loop, agent_loop_continue}` imports exist only as compile-time witnesses pinned by `_keep_agent_loop_live` ([line 98](../../../../../../modules/crates/server/src/platform/sessions/launch.rs#L98)).
- **Classification**: `contradicts-concept`
- **phi-core leverage status**: `direct-reuse` — `phi_core::agent_loop` is invoked at runtime from `spawn_agent_task` ([`launch.rs`](../../../../../../modules/crates/server/src/platform/sessions/launch.rs)). `_keep_agent_loop_live` compile-time witness preserved as belt-and-braces protection against future phi-core renames. (Was `leverage-violation` pre-CH-02.)

## Plan vs. reality
- **Plan said** (plan-archive line 1254, §Part 4 P4 Step 7): *"Spawn `tokio::task` running `phi_core::agent_loop(prompts, ctx, cfg, tx, cancel_token)`. The event channel feeds `BabyPhiSessionRecorder`."*
- **Reality (post-CH-02 seal at current HEAD)**: `spawn_agent_task` (renamed from `spawn_replay_task`) at [`launch.rs`](../../../../../../modules/crates/server/src/platform/sessions/launch.rs) calls `phi_core::agent_loop()` on a spawned tokio task. Events stream through an unbounded mpsc channel into `BabyPhiSessionRecorder::on_phi_core_event`; agent_loop + drain run concurrently via `tokio::join!` on the same task. At M5 the provider is `MockProvider::text(profile.mock_response.unwrap_or("Acknowledged."))` per ADR-0032; real providers defer to M7's credentials-vault milestone.
- **Root cause**: `cascading-upstream-deferral` (provider credentials + tool impls both out of scope → synthetic was least-bad compromise to prove the compound-tx chain works). **Resolved by CH-02** decoupling provider concerns: MockProvider runs deterministically without credentials, so phi-core integration could land before M7 ships the credentials vault.

## Remediation scope (estimate only)
- **Approach (sketch)**: Strip `spawn_replay_task`. Build `AgentContext` from `SessionLaunchContext` + first prompt. Instantiate `phi_core::MockProvider` (deterministic, no network) as the `StreamProvider` via `AgentLoopConfig.provider_override`. Wire `mpsc::unbounded_channel` + event-consumer task that drains the channel and calls `recorder.on_phi_core_event`. Call `agent_loop(prompts, &mut ctx, &cfg, tx, cancel_token).await` in the spawned task. After loop returns, call `recorder.finalise_and_persist`. Real providers defer to M7+.
