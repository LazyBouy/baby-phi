## Concept alignment
- **Concept doc(s)**: [`concepts/phi-core-mapping.md`](../../../concepts/phi-core-mapping.md) ¬ß"wrap: baby-phi field holds phi-core type"
- **Concept claim (verbatim or close paraphrase)**: Baby-phi wraps phi-core types by holding them as a field (`inner: phi_core::X`). The wrap extends with phi-only governance fields alongside the phi-core value. Serde mechanics (flatten vs nested) are not specified at the concept layer.
- **Contradiction**: None. The shipped nested-`inner` form matches the concept-mapping "wrap: baby-phi field holds phi-core type" verbatim. The original plan's `#[serde(flatten)]` directive would have broken this by destructuring phi-core's fields into the parent shape.
- **Classification**: `concept-silent-plan-filled-gap` (serde form is below concept granularity; shipped choice honors the wrap principle)
- **phi-core leverage status**: `wrap` ‚Äî three wrap types (`Session.inner: PhiCoreSession`, `LoopRecordNode.inner: PhiCoreLoopRecord`, `TurnNode.inner: PhiCoreTurn`) compile-time-witnessed in `nodes.rs` test module.

## Plan vs. reality
- **Plan said** (plan-archive line 1132, ¬ßPart 1.5 + ¬ßG3): Session wrap code block carried `#[serde(flatten)] pub inner: phi_core::session::model::Session` on all three wraps, described as "M3 `OrganizationDefaultsSnapshot` pattern byte-for-byte".
- **Reality (shipped state at current HEAD)**: All three wraps use plain nested `pub inner: PhiCoreSession` / `PhiCoreLoopRecord` / `PhiCoreTurn` with no `#[serde(flatten)]` ‚Äî [`nodes.rs:841-926`](../../../../../../modules/crates/domain/src/model/nodes.rs#L841-L926). Wire shape: `{"id": "...", "inner": {"session_id": "...", ...}, "owning_org": "...", ...}`. Two blockers drove the change: (1) phi-core's `Session` already carries `session_id: String` which would collide with baby-phi's `id: SessionId` on flatten; (2) the cited M3 precedent (`OrganizationDefaultsSnapshot`) actually uses nested form, not flatten.
- **Root cause**: `reality-feedback-contradicted-estimate` (plan cited M3 precedent inaccurately; phi-core field collision discovered at implementation).
