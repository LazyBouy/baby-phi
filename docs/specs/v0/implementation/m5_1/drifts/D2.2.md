## Plan vs. reality
- **Plan said** (plan-archive line 1173, §P4 §launch_session compound tx): `RELATE session -> runs_session -> project`. Natural implementation: inline `RELATE type::thing('session', $sid)->runs_session->type::thing('project', $pid)`.
- **Reality (shipped state at current HEAD)**: SurrealDB's RELATE parser rejects inline `type::thing(...)` in FROM/TO slots with "Unexpected token `::`, expected :". The LET-first idiom is required: `LET $f = type::thing('session', $sid); LET $t = type::thing('project', $pid); RELATE $f -> runs_session -> $t SET id = type::thing('runs_session', $edge) RETURN NONE` ([`repo_impl.rs:2256-2259`](../../../../../../modules/crates/store/src/repo_impl.rs#L2256-L2259)). Pattern matches pre-existing M1/M2 edge-writer helpers (`upsert_ownership_raw` etc.).
- **Root cause**: `reality-feedback-contradicted-estimate` (SurrealDB parser constraint not mentioned in existing M1/M2 helper comments loud enough for the M5 planner to notice).
