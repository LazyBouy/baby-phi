## Concept alignment
- **Concept doc(s)**: [`concepts/agent.md`](../../../concepts/agent.md) §"agent lifecycle — active, disabled, archived" + [`concepts/system-agents.md`](../../../concepts/system-agents.md) §"operator can disable — pauses trigger subscriber"
- **Concept claim (verbatim or close paraphrase)**: Agents carry a durable lifecycle state — active, disabled, archived. Disable pauses the trigger subscriber. Archive removes the agent from live operation.
- **Contradiction**: **Contradicts concept directly.** `Agent` node at [`nodes.rs:187`](../../../../../../modules/crates/domain/src/model/nodes.rs#L187) has no `active: bool` or `archived_at: Option<DateTime<Utc>>` columns. Disable + archive handlers emit audit events and return 200 but do NOT mutate any durable state — the concept's "active vs disabled vs archived" distinction does not exist in persistence.
- **Classification**: `contradicts-concept`
- **phi-core leverage status**: `N/A — no phi-core overlap` (agent-lifecycle is baby-phi governance)

## Plan vs. reality
- **Plan said** (plan-archive line 1370, §Part 4 P6 + R-ADMIN-13-W3/W4): disable pauses the trigger subscriber + "marks the agent `active: false`"; archive = "graph archival".
- **Reality (shipped state at current HEAD)**: `Agent` struct has `{id, kind, display_name, owning_org, role, created_at}` — no active/archived_at ([`nodes.rs:187-201`](../../../../../../modules/crates/domain/src/model/nodes.rs#L187-L201)). Migration 0005 did not add these columns. Disable handler ([`disable.rs`](../../../../../../modules/crates/server/src/platform/system_agents/disable.rs)) emits audit + returns 200; archive handler ([`archive.rs`](../../../../../../modules/crates/server/src/platform/system_agents/archive.rs)) gates on standards + emits audit. Neither writes to the agent row.
- **Root cause**: `cascading-upstream-deferral` (migration 0006 would need to add columns; deferred past M5 because "migration 0005 was big enough" — load-bearing deferral that now must close).

## Remediation scope (estimate only)
- **Approach (sketch)**: Migration 0006 adds `ALTER TABLE agent ADD FIELD active DEFAULT true TYPE bool; ALTER TABLE agent ADD FIELD archived_at TYPE option<datetime>;`. Extend `Agent` struct with `#[serde(default = "true_default")] pub active: bool` + `#[serde(default)] pub archived_at: Option<DateTime<Utc>>`. Add 2 repo methods: `flip_agent_active(agent_id, active)` + `set_agent_archived_at(agent_id, at)`. Wire handlers to call them. `#[serde(default)]` guarantees back-compat with pre-migration rows.
