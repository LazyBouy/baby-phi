## Plan vs. reality
- **Plan said** (plan-archive line 1270, §Part 4 P4 §agents/update.rs flip): *"Persist."* — implying existing `upsert_agent_profile` handles both create + update.
- **Reality (shipped state at current HEAD)**: SurrealDB `upsert_agent_profile` uses `UPDATE type::thing('agent_profile', $id) CONTENT $body` — silently no-ops on non-existent rows. The C-M5-5 change arm creates fresh profile rows for agents that had none; `upsert` dropped the write. Fix at [`update.rs:358-366`](../../../../../../modules/crates/server/src/platform/agents/update.rs#L358-L366): branches on `current_profile.is_some()` → `upsert_agent_profile` when prior row exists, `create_agent_profile` otherwise.
- **Root cause**: `reality-feedback-contradicted-estimate` (same SurrealDB UPDATE-on-nonexistent silent-noop as D2.1).
