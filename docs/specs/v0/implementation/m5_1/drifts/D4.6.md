## Plan vs. reality
- **Plan said** (plan-archive line 1286, §P4 Step 6 + §P3 recorder's `finalise_and_persist`): Session + first LoopRecord persisted, then recorder finalises — no clarification of what happens if both try to write the session row.
- **Reality (shipped state at current HEAD)**: Without the fix, recorder's `finalise_and_persist` calls `persist_session` a second time after the launch chain already wrote the row; CREATE path hits UNIQUE constraint and fails. Fix at [`session_recorder.rs:76`](../../../../../../modules/crates/domain/src/session_recorder.rs#L76) (`first_loop_id: Option<LoopId>`) + branch at [line 255](../../../../../../modules/crates/domain/src/session_recorder.rs#L255) (`if self.ctx.first_loop_id.is_some()` → append-only path; otherwise full-persist path for standalone tests).
- **Root cause**: `underspecified-plan` (coordination between launch-chain persist and recorder finalise was not explicit).
