- **Reality (shipped state at current HEAD)**: `inner: Arc<Mutex<PhiCoreSessionRecorder>>` at [`session_recorder.rs:92`](../../../../../../modules/crates/domain/src/session_recorder.rs#L92). Driver: `phi_core::SessionRecorder::on_event(&mut self, ...)` requires mutable self; sharing the recorder across tasks in an `AppState::session_registry` requires `Send + Sync`; `Arc<Mutex<_>>` is the smallest available shape. `started_emitted: Arc<Mutex<bool>>` uses the same pattern to dedupe `SessionStarted` emission ([line 100](../../../../../../modules/crates/domain/src/session_recorder.rs#L100)). `on_phi_core_event(&self, ...)` takes `&self` (not `&mut self`), routes through `self.inner.lock()`, and releases before any `.await` (clippy `await_holding_lock` enforced).
- **Root cause**: `reality-feedback-contradicted-estimate` (phi-core's `&mut self` API + Send+Sync sharing need wasn't anticipated in the P0 plan's minimal sketch).
