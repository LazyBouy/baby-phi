## Plan vs. reality
- **Plan said** (plan-archive line 1313, by omission): M4's Template A fire event lives in `domain::audit::events::m4::templates::template_a_grant_fired`. Implied convention: page 12's audit events would also live under `domain::audit::events::m5::templates`.
- **Reality (shipped state at current HEAD)**: Page 12's audit events (`template.adopted` / `adoption_denied` / `revoked`) are platform-tier events emitted by server handlers — live in [`server/src/platform/templates/audit_events.rs`](../../../../../../modules/crates/server/src/platform/templates/audit_events.rs) (4 builder fns + 4 unit tests). Domain-tier events (fire listeners, state-machine transitions) stay at `domain::audit::events::mX::*`. Convention: platform-handler events at `server::platform::<page>::audit_events`; domain-tier events at `domain::audit::events::mX::*`.
- **Root cause**: `implementer-unilateral-choice` (cleaner separation between handler events and listener events with distinct provenance — adoption AR vs grant id).
